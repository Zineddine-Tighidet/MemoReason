📖 Document Annotation Guide

📋 Annotation Mission

Your job is to deliver a clean, consistent final annotation for each document.

You are reviewing an AI agent draft: start from existing AI annotations, then correct, complete, or remove them as needed.

  1. Check span boundaries: annotate the exact text, no extra words or punctuation.
  2. Pick the right entity type + attribute: use the most specific option available.
  3. Reuse entity IDs for repeated mentions: if two spans refer to the same entity, they must share the same ID. Example: Barack Obama, Obama, and he should all use person_1; a different person must be person_2.
  4. Review questions and rules: they must align with document annotations.

Quality target: every important entity mention is captured, typed correctly, and linked consistently.

🧭 Taxonomy & Attributes

One card = one entity type. Click a card to see all its attributes and descriptions.

Canonical taxonomy spec: TAXONOMY.md.

👤 person — Individual human being mentioned in the document
  • full_name: Complete name of the person (first, middle, and/or last).
  • first_name: Person's given name.
  • middle_name: Person's middle name.
  • last_name: Person's family name.
  • age: Person's age in years.
  • gender: Person's gender.
  • nationality: Person's nationality or citizenship.
  • ethnicity: Person's ethnic or cultural background.
  • subj_pronoun: Subject pronoun (he/she/they).
  • obj_pronoun: Object pronoun (him/her/them).
  • poss_det_pronoun: Possessive determiner (his/her/their).
  • poss_pro_pronoun: Possessive pronoun (his/hers/theirs).
  • refl_pronoun: Reflexive pronoun (himself/herself/themselves).
  • honorific: Title or honorific.
  • relationship.person_Y: Relationship attribute between people.
📍 place — Geographic location or landmark
  • city: Name of a city or town.
  • region: Geographic region or area.
  • state: State or province.
  • country: Nation or country.
  • street: Street name or address.
  • natural_site: Natural landmark or feature.
  • continent: A continent mention.
  • demonym: Demonym linked to a place (French, Syrian, Japanese).
🎯 event — Named event or occurrence
  • name: Name of the event.
  • type: Type or category of event.
🛡️ military_org — Military organization
  • name: Name of the military organization.
🏢 entreprise_org — Private/company organization (enterprise)
  • name: Name of the enterprise organization.
🤝 ngo — NGO / non-governmental organization
  • name: Name of the NGO.
🏛️ government_org — Government institution or agency
  • name: Name of the government organization.
🎓 educational_org — Educational institution
  • name: Name of the educational organization.
📰 media_org — Media/journalism organization
  • name: Name of the media organization.
📅 temporal — Time-related information (dates, times, etc.)
  • day: Day of the week.
  • date: Full date.
  • year: Year.
  • month: Month name or number.
  • timestamp: Specific time or timestamp.
  • day_of_month: Day number within a month.
🔢 number — Numerical values and quantities
  • int: Integer number.
  • str: Number expressed as text.
  • float: Decimal number.
  • fraction: Fractional value.
  • percent: Percentage value between 0 and 100.
  • proportion: Proportion value between 0.0 and 1.0.
🏆 award — Entity representing an award
  • name: Award name mention.
⚖️ legal — Legal or regulatory instrument
  • name: Law/directive/regulation/treaty/policy framework name.
  • reference_code: Official legal citation or identifier.
📦 product — Product entity
  • name: Product name as mentioned in text.

✅ High-Value Checks

  • Names: choose the attribute that matches the exact selected span: full mention = full_name, given name only = first_name, family name only = last_name, middle name only = middle_name.
  • Pronouns: annotate person pronouns (subj_pronoun, obj_pronoun, poss_*, refl_pronoun).
  • Demonyms: words like "French", "Syrian", "Japanese" are usually place_X.demonym.
  • Numbers: use percent for 0-100 values, proportion for 0.0-1.0 values.
  • Legal references: annotate regulation names as legal.name and official codes as legal.reference_code.

🛠️ Editing Workflow

  • Add: select text then annotate.
  • Edit type/attribute: double-click the span.
  • Resize: drag the left/right handles to fix boundaries.
  • Delete: click the × on a span.
  • Consistency check: click entities in the left sidebar to highlight repeated mentions.

QA Annotation Guide

1) Question Types

  • Extractive: this type of question asks for an answer that is directly available in the document and the task is mainly finding and extracting it.
  • Arithmetic: the answer to this question must require arithmetic operations on non-temporal quantities (e.g. "What is the total number of awards Messi won during his carrier?" -- sum over all the awards).
  • Temporal: answer requires reasoning and arithmetic over temporal entities (years, dates, ages, etc.).
  • Inference: basically a general reasoning question that is not Arithmetic, Temporal, or Extractive.

2) Answer Types

  • Variant: answer depends on replaced entities (i.e. annotated entities), so it changes between factual and fictional settings.
  • Invariant: answer comes from non-replaced entities (i.e. non-annotated text spans) and stays the same across settings (factual & fictional).
  • Refusal: the document does not determine the answer (default answer is "Cannot be determined").

3) Coverage Requirement Before Submit

You must cover all combinations: 4 question types × 3 answer types = 12 combinations.

If one combination is impossible in a document (for example no numeric values for arithmetic), use Ignore justification in the coverage verifier to explain why it is impossible to have the ignored type of question.

4) Special Note: Temporal + Invariant

Temporal + Invariant questions are often hard because most temporal mentions are annotated and replaced.

If there is no usable non-annotated temporal mention, write a temporal comparison question about event ordering (for example, “Which happened first?”). Our transformations preserve temporal order between factual and fictional versions.

5) Reasoning Chain (Optional)

You can add reasoning steps to explain how to answer the question. Write one step per line. Keep steps short and document-grounded.
